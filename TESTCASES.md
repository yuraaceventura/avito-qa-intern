Cоздание объявления (ручка /api/1/item)  
1.   
Название: Успешное создание объявления с валидными данными  
Предусловия: -  
Шаги:  
    1. Отправить POST запрос на /api/1/item с валидными данными  
    2. Проверить ответ  
Ожидаемый результат:  
    1. Ответ возвращается со статусом 200 - Created  
    2. В теле ответа содержится сгенерированный uuid объявления  
  
2.   
Название: Создание объявления без обязательных полей  
Предусловия: -  
Шаги:  
    1. Отправить POST запрос на /api/1/item, не указав одно или несколько обязательных полей   
    2. Проверить ответ  
Ожидаемый результат:  
    1. Ответ возвращается со статусом 400 - Bad Request  
    2. В теле ответа содержится поле message, с сообщением об отсутствии обязательного поля  
  
3.   
Название: Создание объявления с неверным типом данных  
Предусловия: -  
Шаги:  
    1. Отправить POST запрос на /api/1/item, указав неверный тип данных у одного или нескольких полей   
    2. Проверить ответ  
Ожидаемый результат:  
    1. Ответ возвращается со статусом 400 - Bad Request  
    2. В теле ответа содержится поле message, с сообщением о неверном теле запроса  
  
4.   
Название: Создание объявления с негативными значениями полей price, likes, viewCount и contacts  
Предусловия: -  
Шаги:  
    1. Отправить POST запрос на /api/1/item с негативными значениями полей price, likes, viewCount и contacts  
    2. Проверить ответ  
Ожидаемый результат:  
    1. Ответ возвращается со статусом 400 - Bad Request  
    2. В теле ответа содержится поле message, с сообщением о невалидном теле запроса  
  
5.   
Название: Создание дубликата объявления  
Предусловия: -  
Шаги:  
    1. Отправить POST запрос на /api/1/item с валидными данными   
    2. Проверить ответ  
    3. Повторить запрос из пункта 1  
    4. Проверить ответ  
Ожидаемый результат:  
    1. Первый запрос возвращает статус 200 - OK и uuid объявления  
    2. Второй запрос возвращает статус 200 - OK и uuid объявления, отличный от uuid первого объявления (или возвращает код 409 Conflict)

Общие проверки  
  
1.   
Название: Проверца целостности данных после создания объявления  
Предусловия: -  
Шаги:  
    1. Отправить POST запрос на /api/1/item с валидными данными  
    2. Отправить GET запрос на /api/1/item/{id} c id полученным в пункте 1  
    3. Сравнить данные  
Ожидаемый результат:  
    1. Данные отправленные в шаге 1 соответствуют данным полученным в шаге 2  
    2. В поле created_at стоит корректная дата создания объявления  
  
  
2.   
Название: Проверка логики ведения статистики  
Предусловия: -  
Шаги:  
    1. Отправить POST запрос на /api/1/item с валидными данными, указать viewСount = 0  
    2. Отправить GET запрос на /api/1/item/{id} c id полученным в пункте 1  
    3. Проверить ответ  
Ожидаемый результат:  
    1. Запрос из пункта 2 вернул корректные данные  
    2. Поле viewСount увеличилось на 1
